## 🐘 분산 컴퓨팅 개요
---
분산 컴퓨팅은 다수의 컴퓨터가 공통의 문제를 해결하기 위해 협업하도록 만드는 방법입니다.

보통 병렬 컴퓨팅이랑, 분산 컴퓨팅을 헷갈려 하는 사람들이 있는데, 둘은 분명 다른 이론입니다.

### 🐘 병렬 컴퓨팅 vs 분산 컴퓨팅

> 💡 **병렬 컴퓨팅과 분산 컴퓨팅, 둘은 다릅니다!**

| **구분** | **병렬 컴퓨팅 (Parallel Computing)** | **분산 컴퓨팅 (Distributed Computing)** |
|--------|-------------------------------------------------------|-------------------------------------------------------|
| **구조** | 하나의 컴퓨터 내부에서 여러 자원 (GPU, CPU 코어 등) 활용 | 여러 개의 독립된 컴퓨터(노드)가 네트워크로 연결 |
| **작업 방식** | 하나의 큰 작업을 여러 작은 작업으로 나누어 **동시에 처리** | 여러 노드가 협업하여 하나의 작업을 **분산** 처리 |
| **메모리** | **공유 메모리 (RAM)** 사용 | 각 노드가 **개별 메모리** 사용, 데이터는 네트워크로 이동 |
| **예시** | 딥러닝 모델 학습, AI 처리, GPU 병렬 처리 | 하둡(Hadoop), 스파크(Spark), 페이스북 데이터 처리 등 |

<br>

### 🐘 병렬 컴퓨팅 예시

- 딥러닝 모델 학습 시, 하나의 컴퓨터에 여러 GPU, CPU 코어를 두고 **공유 메모리**로 데이터를 나누어 처리
- 8K 해상도의 이미지를 여러 블록으로 쪼개어 여러 GPU 코어에서 동시에 처리
- **주요 특징:** 하나의 컴퓨터 내에서 빠른 처리, 데이터 이동 거의 없음

---

### 🐘 분산 컴퓨팅 예시

- **하둡**과 같은 대규모 데이터 처리 플랫폼
- 페이스북처럼 하루 수억 건의 데이터가 생성되는 시스템
- 하나의 컴퓨터로 처리 불가능 → 여러 대의 컴퓨터에 작업을 **분산**
- **주요 특징:** 대규모 데이터 처리에 적합, 서버 장애 시에도 데이터 손실 최소화, 유연한 확장성

<br>

## 🐘 분산 컴퓨팅 특징
---

분산 컴퓨팅의 주요 특징과 장점을 좀 더 깊게 들여다보겠습니다.

### 1️⃣ 확장성 (Scalability)
- 워크로드에 따라 유연하게 확장 가능
- **필요 시 노드 추가**로 컴퓨팅 성능 증대

---

### 2️⃣ 가용성 (Availability)
- 일부 노드가 장애 발생해도 시스템은 정상 동작
- **내결함성(Fault Tolerance)** 기본 내장

---

### 3️⃣ 일관성 (Consistency)
- 데이터 중복, 분산 저장에도 **데이터 일관성 유지**
- 시스템 전반에 걸쳐 데이터 싱크 맞춤

---

### 4️⃣ 투명성 (Transparency)
- 사용자에게는 **마치 하나의 컴퓨터처럼 동작**
- 물리적 인프라의 복잡성을 숨김

---

### 5️⃣ 효율성 (Efficiency)
- 리소스 최적화, 고성능 제공
- 시스템 장애, 고가 하드웨어 비용 문제 최소화

**분산 컴퓨팅에는 이렇게 다섯 가지의 큰 장점이 있다고 볼 수 있겠습니다.**


## 🐘 CAP 이론
---
분산 컴퓨팅 하면 **꼭 짚고 넘어가야 하는 중요한 이론**이 있습니다.

바로 **CAP 이론**입니다.

![Image](https://github.com/user-attachments/assets/f8fbffe6-4dee-4025-aac6-331a2589227a)

분산 컴퓨팅, 분산 스토리지와 관련 된 공부를 할 때 꼭 알아야 하는 기초 이론 중 하나입니다.

분산 컴퓨팅 환경은 물리적인 별개의 노드들로 하나의 시스템을 구성하기 위해 네트워크를 통해 서로 통신하고 싱크를 맞추게 되는데, 수평 확장이 가능하며, 비용 효율을 얻을 수 있고, fault tolerant 한 시스템을 구축 할 수 있다는 장점이 있지만, 결국 여러 노드들이 동작하는 물리적 환경으로 인해 어쩔 수 없이 발생하는 한계점이 있을 수 밖에 없습니다. CAP 이론은 이런 분산 환경의 성질을 크게 3가지로 나눠 설명하고, 그 안에서 어떤 서비스를 구축할 것인가에 따라 어떤 부분에 중점을 두어야 하는지를 논하는 것이 핵심입니다.

왜냐하면, CAP 이론은 C, A, P 모두 만족 할 수 없으며, 이 안에서 최대 두 가지에 집중해야 한다는 이론이기 때문입니다.

CAP 를 각각 나눠서 봅시다.

| **속성** | **설명** |
|--------|---------------------------------------------------|
| **C (Consistency)** | 일관성, 모든 노드에서 동일한 데이터 보장 |
| **A (Availability)** | 가용성, 모든 요청에 대해 항상 응답 제공 |
| **P (Partition Tolerance)** | 파티션 허용, 네트워크 분할 상황에서도 시스템 지속 동작 |

- **Consistency (일관성)**  
  모든 노드에서 **항상 동일한 데이터** 보장

- **Availability (가용성)**  
  시스템은 항상 **응답**해야 함

- **Partition Tolerance (파티션 허용)**  
  네트워크 장애나 노드 간 **통신 실패 발생 시에도 시스템 지속 운영**
